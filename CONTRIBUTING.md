# Contributing

Thanks for your interest in contributing!

## Ways to contribute

- Bug reports and feature requests via Issues
- Documentation improvements
- Code changes via Pull Requests

## Pull Request process

1. Fork the repository and create your branch from `main`.
2. Keep changes focused and include tests when reasonable.
3. Update documentation when behaviour changes.
4. Ensure CI passes.
5. Open a Pull Request with a clear description of the change and motivation.

## Developer Certificate of Origin (DCO)

By contributing to this project, you agree to the Developer Certificate of Origin (DCO)
included in the `DCO` file.

### Sign-off your commits

All commits must be signed off. Add the `-s` flag to your commits:

```bash
git commit -s -m "feat: add X"
